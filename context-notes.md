# 로그인/회원가입/마이페이지 편의성 개선 및 에러 처리 관련 컨텍스트 노트

## 결정 사항

- **비밀번호 규칙 실시간 안내 방식**
  회원가입 화면 및 마이페이지 비밀번호 변경 화면에서 비밀번호 입력창 바로 아래에 가이드라인 문구를 상시 표시합니다. 사용자가 입력할 때 정규식 만족 여부를 실시간으로 판별하여, 통과 시 초록색 텍스트와 체크 표시(✓)를 활성화하여 실시간 피드백을 극대화합니다.

- **로그인 화면 규칙 안내 미표시**
  로그인 화면은 이미 등록된 계정으로 로그인하는 영역이므로 복잡한 비밀번호 정규식 규칙 안내를 노출하지 않기로 결정했습니다. 대신 평문 토글 기능만 추가하여 사용자 입력 편의를 돕습니다.

- **비밀번호 확인 실시간 일치 안내**
  회원가입의 비밀번호 확인창 및 마이페이지의 새 비밀번호 확인창 아래에는 규칙 대신 비밀번호와 일치하는지 여부를 실시간으로 보여줍니다. 일치할 경우 초록색 체크 상태와 함께 일치 안내 문구를 띄워 제출 전에 확인 가능하도록 합니다.

- **에러 코드 상수화**
  프론트엔드와 백엔드의 동일 리포지토리 이점을 활용하기 위해 `src/lib/error-codes.ts` 파일에 에러 코드 상수를 공유 정의합니다. 백엔드의 응답 객체와 프론트엔드의 토스트 처리기에 이 공통 타입을 매핑하여 타입 미스매치로 인한 버그를 미연에 방지합니다.

- **서버 에러 대응 토스트 매핑**
  - `MISSING_FIELDS`: "필수 입력 항목을 모두 작성해주세요."
  - `INVALID_CREDENTIALS`: "아이디 혹은 비밀번호가 틀립니다."
  - `DUPLICATE_USERNAME`: "이미 사용 중인 아이디입니다."
  - `DUPLICATE_NICKNAME`: "이미 사용 중인 닉네임입니다."
  - `UNAUTHORIZED`: "로그인이 필요합니다."
  - `USER_NOT_FOUND`: "사용자를 찾을 수 없습니다."
  - `INVALID_PASSWORD`: "기존 비밀번호가 일치하지 않습니다."
  - `SERVER_ERROR` 및 기타 예외: "서버에 일시적인 문제가 발생했습니다."
  - 네트워크 장애: "서버와 통신할 수 없습니다. 네트워크 상태를 확인해주세요."
