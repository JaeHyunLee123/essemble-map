# 테스트 개발 계획 (Test Strategy)

이 문서는 합주실 지도 애플리케이션의 안정성과 품질을 보장하기 위한 테스트 환경 구축 및 작성 계획을 다룹니다.

## 1. 테스트 목적 및 도구
*   **목적:** 지도 렌더링, 제보 프로세스, 북마크 등의 핵심 비즈니스 로직과 UI 인터랙션을 자동화된 테스트로 검증하여 회귀 버그를 예방합니다.
*   **도구:**
    *   **Unit/Integration Test:** `Vitest` (빠른 실행 속도와 Vite/Next.js 호환성 활용)
    *   **E2E Test:** `Playwright` (실제 브라우저 환경에서의 지도 상호작용 및 사용자 플로우 테스트)

## 2. 단위 및 통합 테스트 (Vitest)
*   **유틸리티 함수 검증:** 좌표 계산, 데이터 파싱, 토큰 검증 로직 등 순수 함수 형태의 비즈니스 로직 테스트.
*   **UI 컴포넌트 렌더링:** 모달 창, 제보 폼, 마커 컴포넌트 등이 주어진 데이터에 맞게 올바르게 렌더링되는지 확인 (React Testing Library 병행 사용).
*   **API Route Handlers:** Mock Request를 통해 인증 미들웨어 및 데이터베이스 조작 로직(Supabase/Drizzle)이 올바르게 동작하는지 테스트.

## 3. E2E 테스트 (Playwright)
실제 사용자 관점에서의 핵심 시나리오를 검증합니다.
*   **지도 및 클러스터링 로직:** 지도 이동 및 확대/축소 시 클러스터링이 올바르게 계산되고 마커가 다시 렌더링되는지 확인.
*   **사용자 인증 플로우:** 회원가입, 로그인 및 마이페이지 접근 권한 제어 검증.
*   **합주실 제보 및 승인 플로우:**
    1.  일반 유저의 합주실 제보(이미지 첨부 등) 정상 등록 확인.
    2.  어드민 계정 로그인 후 대기열 확인 및 승인(`active` 상태 변경) 프로세스 검증.

## 4. CI/CD 파이프라인 통합
*   GitHub Actions를 연동하여 PR 생성 및 메인 브랜치 병합 시 자동으로 코드 린팅(ESLint) 및 Vitest/Playwright 테스트가 실행되도록 파이프라인을 구축합니다.
