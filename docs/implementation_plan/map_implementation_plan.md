# 네이버 지도 및 클러스터링 (Map) 구현 계획

이 문서는 수천 개 이상의 마커를 효율적으로 렌더링하기 위한 클라이언트 지도 구성과 서버 API 전략을 다룹니다.

## 1. 지도 뷰포트 API 쿼리
*   **API 구현 (`GET /api/studios/map`):**
    *   쿼리 파라미터로 지도의 북동쪽 좌표(neLat, neLng)와 남서쪽 좌표(swLat, swLng)를 받습니다.
    *   DB에서 해당 `Bounding Box` 범위 내에 존재하는 합주실만 필터링합니다.
    *   클라이언트 렌더링에 필요한 최소한의 데이터(id, name, lat, lng)만을 JSON으로 응답합니다.

## 2. 프론트엔드 최적화 및 상태 관리
*   **이벤트 디바운싱:** 유저가 지도를 드래그(Pan)하거나 줌(Zoom)할 때 즉시 API를 호출하지 않고, 동작이 멈춘 후 약 300ms 뒤에 한 번만 요청하도록 최적화합니다.
*   **서버 상태 캐싱:** `TanStack Query`의 `useQuery`를 사용하며, 좌표 영역 또는 줌 레벨을 쿼리 키(Query Key)의 일부로 사용하여 기존에 방문했던 지역의 캐시 히트율을 높입니다.

## 3. 마커 클러스터링 (Marker Clustering)
*   지도에 표시할 데이터가 많을 때 발생하는 브라우저 버벅임(Lag)과 시각적 복잡도를 해소합니다.
*   클라이언트단에서 `supercluster` 라이브러리를 활용하거나 네이버 지도의 공식 클러스터링 플러그인을 래핑하여 구현합니다.
*   줌 레벨에 따라 근접한 마커들을 하나로 합쳐서 숫자 표기(예: '120')로 렌더링하고, 클릭 시 해당 영역으로 확대되도록 상호작용을 추가합니다.
*   **동일 좌표 처리:** 최대로 확대한 상태에서도 같은 위치(건물 등)에 2개 이상의 합주실이 존재할 경우(오버랩), 마커 클릭 시 모달이나 바텀 시트로 해당 위치의 "합주실 목록"을 표시하는 UI를 제공하여 접근성을 보장합니다.
