# 데이터 크롤러 및 적재 (Crawler) 구현 계획

이 문서는 서비스 초기 운영을 위한 합주실 데이터 수집 스크립트의 구성 및 실행 흐름을 다룹니다.

## 1. 프로젝트 통합 (Monorepo 방식)
*   Next.js 프로젝트 내부의 `scripts/` 디렉터리에 크롤러 코드를 위치시킵니다.
*   `ts-node`나 `tsx`를 사용하여 스크립트를 독립적으로 실행할 수 있게 구성합니다.
*   웹 프로젝트와 동일한 Drizzle 스키마를 Import하여 DB 조작의 일관성을 유지합니다.

## 2. Playwright 설정 및 스크래핑
*   Playwright를 통해 네이버 지도의 합주실 검색 결과 페이지를 크롤링합니다.
*   스크롤 또는 페이지네이션을 통해 수집 대상 합주실 리스트를 확보합니다.
*   각 합주실의 상세 페이지에서 상호명, 위치(위도/경도), 사진 URL, 가격 정보, 장비 목록을 추출합니다.

## 3. 이미지 다운로드 및 업로드 파이프라인
*   웹상의 원본 이미지 URL들을 직접 DB에 넣지 않습니다.
*   Node.js의 `axios`나 `fetch`를 사용해 이미지 버퍼(ArrayBuffer)를 메모리에 다운로드합니다.
*   해당 버퍼를 `@supabase/supabase-js` Storage API를 사용해 업로드합니다.
*   생성된 Supabase 공개 URL을 최종 데이터베이스 모델 객체에 매핑합니다.

## 4. DB 일괄 삽입 (Seeding)
*   수집 및 변환이 완료된 데이터 뭉치를 Drizzle ORM의 `insert().values().onConflictDoNothing()` 등의 메서드를 활용해 배치로 DB에 적재합니다.
*   데이터 삽입 시 합주실 상태는 `active`로 고정하여 지도에 즉시 표시되도록 합니다.
