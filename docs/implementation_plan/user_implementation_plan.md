# 마이페이지 및 유저 기능 (User) 구현 계획

이 문서는 유저의 개인정보 수정과 활동 내역(북마크, 제보)을 조회하는 마이페이지 기능 구현을 다룹니다.

## 1. 개인정보 수정 API
*   **닉네임 변경 (`PATCH /api/user/profile`):**
    *   요청받은 새로운 닉네임의 중복 여부를 검사합니다.
    *   중복이 아닐 경우 DB의 `Users` 테이블을 업데이트하고 새 유저 정보를 반환합니다.
*   **비밀번호 변경 (`PATCH /api/user/password`):**
    *   현재 비밀번호(currentPassword)가 일치하는지 해시 검증을 수행합니다.
    *   일치할 경우 새로운 비밀번호를 해싱하여 업데이트합니다.

## 2. 유저 활동 내역 조회 API
*   **북마크한 합주실 (`GET /api/user/bookmarks`):**
    *   `Bookmarks` 테이블과 `Studios` 테이블을 이너 조인(Inner Join)하여 유저가 저장한 합주실 목록을 반환합니다.
    *   조회 시 `Studios.status = 'active'` 필터 조건을 강력하게 적용하여, 현재 승인 상태인 합주실만 안전하게 반환합니다.
    *   응답 데이터에는 합주실 이름, 위치, 네이버 지도 링크, 위경도 좌표 등이 포함됩니다.

*   **제보한 합주실 (`GET /api/user/submissions`):**
    *   `Studios` 테이블에서 `created_by`가 현재 유저인 데이터를 조회합니다.
    *   어드민 승인 대기 중인 `pending`, 승인 완료된 `active`, 그리고 반려된 `deny` 상태의 제보 데이터를 모두 반환합니다.
    *   제보가 반려(`deny`)된 경우, 반려 사유(`denyReason`) 필드를 함께 포함하여 반환합니다.
    *   향후 방/장비 등 타 도메인 확장에 대비하여 응답 객체 포맷에 `"type": "studio"` 속성을 기본 포함합니다.

## 3. 프론트엔드 UI 및 접근 제어
*   Next.js 16의 파일 규격인 **`proxy.ts`**를 프로젝트 루트(또는 `src/`)에 생성하여 통합 인증 미들웨어(Proxy)를 구성합니다.
*   `proxy.ts`에서 쿠키의 존재 여부 및 토큰 검증을 수행하며, 로그인하지 않은 사용자가 보호된 마이페이지(`/mypage` 등)에 접근할 경우 메인 화면(`/`)으로 안전하게 리다이렉트합니다.
*   마이페이지 UI에서는 탭 구조를 활용하여 '내 정보 수정', '북마크 목록', '제보 내역' 화면을 분리하여 렌더링하며, 제보 내역의 상태(`pending`, `active`, `deny`)를 배지 형태로 제공하여 직관적으로 확인하게 만듭니다.
