# 마이페이지 및 유저 기능 (User) 구현 계획

이 문서는 유저의 개인정보 수정과 활동 내역(북마크, 제보)을 조회하는 마이페이지 기능 구현을 다룹니다.

## 1. 개인정보 수정 API
*   **닉네임 변경 (`PATCH /api/user/profile`):**
    *   요청받은 새로운 닉네임의 중복 여부를 검사합니다.
    *   중복이 아닐 경우 DB의 `Users` 테이블을 업데이트하고 새 유저 정보를 반환합니다.
*   **비밀번호 변경 (`PATCH /api/user/password`):**
    *   현재 비밀번호(currentPassword)가 일치하는지 해시 검증을 수행합니다.
    *   일치할 경우 새로운 비밀번호를 해싱하여 업데이트합니다.

## 2. 유저 활동 내역 조회 API
*   **북마크한 합주실 (`GET /api/user/bookmarks`):**
    *   `Bookmarks` 테이블과 `Studios` 테이블을 조인하여 유저가 저장한 합주실 목록을 반환합니다.
    *   응답 데이터에는 합주실 이름, 위치, 네이버 지도 링크, 상태(`active`) 등이 포함됩니다.

*   **제보한 합주실 (`GET /api/user/submissions`):**
    *   `Studios` 테이블에서 `created_by`가 현재 유저인 데이터를 조회합니다.
    *   어드민 승인 대기 중인 `pending` 상태의 합주실도 본인에게는 보이도록 처리합니다.

## 3. 프론트엔드 UI
*   Next.js의 Proxy를 사용하여 쿠키(Refresh Token) 유무를 검증하고, 로그인된 유저만 마이페이지 등 보호된 라우트에 접근할 수 있도록 접근 제어를 설정합니다.
*   탭 구조를 활용하여 '내 정보 수정', '북마크 목록', '제보 내역' 화면을 분리하여 렌더링합니다.
