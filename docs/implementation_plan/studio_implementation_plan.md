# 합주실 상세 및 제보 (Studio) 구현 계획

이 문서는 개별 합주실의 세부 정보를 보여주는 모달 기능과, 유저가 직접 합주실을 추가할 수 있는 제보 기능에 대한 MVP 구현 계획을 다룹니다.

## 1. 합주실 상세 뷰 API
*   **상세 API (`GET /api/studios/:id`)**
    *   조인 쿼리 없이 `studios` 테이블의 단일 레코드를 조회하여 반환합니다.
    *   프론트엔드의 추후 확장을 대비해 응답 포맷에 `rooms` 필드를 빈 배열(`rooms: []`)로 구성하여 반환합니다.
    *   예상 응답 형식은 다음과 같습니다.
        ```json
        {
          "success": true,
          "data": {
            "id": "uuid-1",
            "name": "낙원 합주실",
            "description": "최고의 시설...",
            "images": [],
            "lat": 37.1234,
            "lng": 127.1234,
            "mapUrl": "https://map.naver.com/p/entry/place/12345",
            "status": "active",
            "rooms": []
          }
        }
        ```

## 2. 클라이언트 UI (상세 모달)
*   마커 또는 클러스터 맵 클릭 이벤트를 트리거로 상세 모달 컴포넌트를 호출합니다.
*   TanStack Query를 사용해 로딩 중엔 스켈레톤 UI를 보여줍니다.
*   합주실의 이름, 설명, 네이버 지도 외부 링크 이동 버튼을 배치합니다. 방 정보나 장비 정보 영역은 UI에서 완전히 숨기거나 렌더링하지 않습니다.
*   로그인된 유저에 한하여 '북마크 토글 버튼'을 렌더링하며, 낙관적 업데이트(Optimistic Update) 기법으로 즉각적인 피드백을 제공합니다.

## 3. 네이버 지도 링크 기반 위경도 자동 파싱 라이브러리
*   **파일 위치**: `src/lib/naverMap.ts`
*   **기능 구성**
    *   **단축 URL 리다이렉트 추적**: 입력받은 주소가 `naver.me` 형식의 단축 URL인 경우, HTTP GET 또는 HEAD 요청을 통해 리다이렉트 최종 목적지 주소(풀 URL)를 찾아옵니다.
    *   **플레이스 ID 추출**: 네이버 플레이스 상세 풀 URL(`map.naver.com/p/entry/place/{placeId}`)에서 정규표현식(`place/(\d+)`)을 활용하여 플레이스 ID를 파싱합니다.
    *   **네이버 플레이스 요약 API 호출**: 파싱한 플레이스 ID를 바탕으로 네이버 공식 모바일/웹 내부 요약 엔드포인트(`https://map.naver.com/v5/api/sites/summary/{placeId}?lang=ko`)에 fetch 요청을 전송합니다.
    *   **위경도 반환**: API 응답 중 경도(`x` -> float) 및 위도(`y` -> float) 좌표를 WGS84 위경도로 변환하여 반환합니다.
    *   **예외 처리 및 유효성 검증**: 네트워크 오류, 존재하지 않는 플레이스 ID, 파싱 실패 등의 상황 발생 시 커스텀 에러(`InvalidNaverMapUrlError` 등)를 던져 API 라우트에서 처리할 수 있도록 설계합니다.

## 4. 합주실 제보 기능 (Submission)
*   **유저 입력 폼 UI**
    *   `react-hook-form`을 이용하여 합주실 이름, 설명(선택), 네이버 지도 링크 세 가지만 입력받습니다.
    *   이미지 첨부 및 관련 UI 영역은 완전히 생략합니다.
*   **서버 처리 API (`POST /api/studios/submit`)**
    *   사용자 입력 데이터(`name`, `description`, `mapUrl`)를 검증합니다.
    *   `mapUrl`이 유효한 네이버 지도 링크인지 확인하고, `naverMap.ts` 라이브러리를 사용해 위도(`lat`)와 경도(`lng`)를 자동으로 파싱합니다.
    *   **파싱 실패 시**: 즉시 `400 Bad Request` 에러를 반환하며 제보 등록을 거부합니다. 클라이언트 UI에서는 사용자에게 "유효한 네이버 지도 링크가 아닙니다." 라는 친절한 안내 메시지를 출력합니다.
    *   **파싱 성공 시**: 파싱된 위경도(`lat`, `lng`) 값을 포함하여 데이터베이스에 새로운 `studios` 레코드를 저장합니다. 이때 `images` 칼럼은 항상 빈 배열(`[]`)로 저장하고, 상태는 검토 대기(`pending`)로 고정하여 어드민 승인 전까지 지도에 노출되지 않도록 제어합니다.
