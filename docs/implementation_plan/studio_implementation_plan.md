# 합주실 상세 및 제보 (Studio) 구현 계획

이 문서는 개별 합주실의 세부 정보를 보여주는 모달 기능과, 유저가 직접 합주실을 추가할 수 있는 제보 기능에 대해 다룹니다.

## 1. 합주실 상세 뷰 API
*   **상세 API (`GET /api/studios/:id`):**
    *   단순한 마커 데이터가 아닌, 해당 합주실에 속한 `Rooms`, `Equipments`, `EquipmentCategories` 정보를 모두 조인하여 가져옵니다.
    *   사진 배열, 방별 시간당 가격, 허용 인원 및 보유 장비 목록을 포함합니다.

## 2. 클라이언트 UI (상세 모달)
*   마커 또는 클러스터 맵 클릭 이벤트를 트리거로 모달 컴포넌트를 호출합니다.
*   TanStack Query를 사용해 로딩 중엔 스켈레톤 UI를 보여줍니다.
*   이미지 캐러셀, 네이버 지도 외부 링크로 이동하기 버튼을 배치합니다.
*   로그인된 유저에 한하여 '북마크 토글 버튼'을 렌더링하며, 낙관적 업데이트(Optimistic Update) 기법으로 즉각적인 피드백을 제공합니다.

## 3. 합주실 제보 기능 (Submission)
*   **유저 폼 제출 로직:** 
    *   `react-hook-form`을 이용해 상호명, 링크, 설명, 사진을 입력받습니다.
    *   사진은 실제 로컬 파일 첨부 방식과 외부 URL 입력 방식을 모두 제공합니다.
*   **서버 처리 API (`POST /api/studios/submit`):**
    *   **파일 방식:** `FormData`로 전달된 파일을 읽어 Supabase Storage에 업로드합니다.
    *   **URL 방식:** 서버에서 해당 URL의 바이너리를 다운로드한 후 Supabase Storage에 다시 업로드합니다.
    *   DB에 데이터를 기록할 때 상태를 `pending`으로 고정하여 어드민 검토 전까지 지도에 노출되지 않게 방지합니다.
