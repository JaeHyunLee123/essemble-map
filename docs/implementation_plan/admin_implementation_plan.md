# 어드민 페이지 구현 계획 (Admin)

이 문서는 애플리케이션의 운영 및 합주실 데이터 관리를 위한 관리자 전용 대시보드 구현 계획을 다룹니다.

## 1. 기능 요구사항 (Requirements)
*   **유저 통계 확인:** 현재 가입된 전체 유저 수를 대시보드 상단에 표시.
*   **합주실 제보 관리:** 일반 사용자가 제보한 합주실 중 수락 대기 중(`pending`)인 목록을 확인.
*   **수락 및 거절 처리:** 대기 중인 합주실 정보를 검토한 후, 상태를 수락(`active`) 또는 거절(`deny`)로 변경.

## 2. 데이터베이스 및 API (Backend)
*   **API 엔드포인트 (`api_specification.md` 참고):**
    *   `GET /api/admin/stats`: 총 유저 수 카운트 쿼리 수행.
    *   `GET /api/admin/studios/pending`: `status = 'pending'` 인 스튜디오 목록 쿼리.
    *   `PATCH /api/admin/studios/:id/status`: 특정 스튜디오의 상태를 `active` 또는 `deny`로 변경.
*   **보안 제어:**
    *   Next.js Middleware를 활용하여 접속한 사용자의 토큰 내 `role`이 `admin`인지 검사합니다.
    *   권한이 없는 유저가 `/admin/*` 라우트나 `/api/admin/*` API에 접근할 경우 `403 Forbidden` 처리 및 홈으로 리다이렉트합니다.

## 3. 프론트엔드 UI (Frontend)
*   어드민 전용 레이아웃을 구성하여 일반 사용자 페이지와 시각적으로 분리합니다.
*   **대시보드 메인 화면:** 전체 유저 수 등 핵심 통계 위젯 렌더링.
*   **대기열 테이블:** 제보된 합주실의 정보(이름, 네이버 지도 링크, 상세 설명 등)를 목록 형태로 표시.
*   **액션 버튼:** 각 행마다 [수락] 및 [거절] 버튼을 배치하여 상태를 직관적으로 변경할 수 있도록 UI 구현. 거절 시에는 반려 사유를 입력하는 모달을 추가로 제공.
