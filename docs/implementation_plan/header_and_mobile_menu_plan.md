# 구현 계획서 (Implementation Plan) - 합주실 제보 버튼 및 모바일 헤더 개편

비로그인 상태에서 합주실 제보 기능 인지도를 높이고, 모바일 환경에서의 헤더 레이아웃 공간 제약을 해결하기 위한 프론트엔드 컴포넌트 개편 계획입니다.

## 1. 개요 및 목표

- **인지도 개선**: 로그인 여부에 상관없이 헤더에 "합주실 제보" 버튼을 항상 노출합니다.
- **매끄러운 UX 제공**: 비로그인 상태에서 "합주실 제보" 혹은 "마이페이지"에 접근할 경우, 로그인 유도 토스트 알림을 띄우고 즉시 로그인 모달을 엶과 동시에 로그인 완료 시 원래 수행하려던 동작(제보 폼 오픈 또는 마이페이지 이동)으로 자동 연계되게 합니다.
- **모바일 대응**: 모바일 뷰(`sm` 해상도 이하)에서는 헤더의 액션과 메뉴들을 햄버거 메뉴를 통해 여는 사이드바(Sheet)로 숨겨 화면 복잡도를 낮춥니다. 사이드바 내에서 모달이 열려야 할 때는 사이드바를 먼저 닫은 후 모달을 띄워 조작성을 확보합니다.

## 2. 관련 파일 및 리소스

- **설치 대상**: `src/components/ui/sheet.tsx` (shadcn UI Sheet)
- **수정 대상**:
  - `src/components/auth/AuthModal.tsx`: 외부 제어 기능 및 로그인 성공 콜백 지원
  - `src/components/Header.tsx`: 헤더 레이아웃 개편 및 상태 연계 조율 로직 추가

## 3. 세부 구현 단계

### 단계 1: UI 컴포넌트 추가
1. 다음 명령어를 실행하여 shadcn UI `Sheet` 컴포넌트를 설치합니다.
   ```bash
   npx shadcn@latest add sheet --yes
   ```
2. 정상적으로 `src/components/ui/sheet.tsx`가 생성되었는지 확인합니다.

### 단계 2: `AuthModal.tsx` 컴포넌트 확장
부모 컴포넌트에서 로그인 모달을 직접 제어할 수 있도록 `open`, `onOpenChange`, `onSuccess`를 인터페이스에 추가합니다.

- **인터페이스 정의**
  ```typescript
  interface AuthModalProps {
    open?: boolean;
    onOpenChange?: (open: boolean) => void;
    onSuccess?: () => void;
  }
  ```
- **제어 로직 구현**
  - 외부에서 `open` 상태가 전달되면 Controlled 방식으로 작동하고, 전달되지 않으면 내부 `useState`를 통해 Uncontrolled 방식으로 하이브리드 지원하도록 구현합니다.
  - 로그인 및 회원가입 폼이 성공했을 때 실행되는 `handleSuccess` 핸들러 내부에서 `onSuccess` prop이 존재하는 경우 이를 함께 실행해 줍니다.
    ```typescript
    const handleSuccess = () => {
      setOpen(false);
      onSuccess?.();
    };
    ```

### 단계 3: `Header.tsx` 컴포넌트 개편
1. **상태 머신 및 변수 정의**
   - 모바일 메뉴(Sheet) 오픈 여부: `isMenuOpen`
   - 제보 폼 모달 오픈 여부: `isSubmitOpen`
   - 로그인 모달 오픈 여부: `isAuthOpen`
   - 로그인 성공 후 연계 작업 유형: `authCallback` (`"submit" | "mypage" | null`)

2. **로그인 연계 조율 로직**
   - **제보하기 클릭 핸들러 (`handleSubmitting`)**:
     - 로그인 유저인 경우: `isSubmitOpen`을 `true`로 설정합니다.
     - 비로그인 유저인 경우: 
       1. 사이드바가 열려 있다면 닫음 (`isMenuOpen = false`)
       2. `toast.error("로그인이 필요한 서비스입니다.")` 표시
       3. 콜백 유형 설정 (`authCallback = "submit"`)
       4. 로그인 모달 엶 (`isAuthOpen = true`)
   - **마이페이지 클릭 핸들러 (`handleMyPageClick`)**:
     - 로그인 유저인 경우: `/mypage`로 정상 이동합니다. (현재 Link로 되어 있으므로, 비로그인 시의 커스텀 핸들러로 동작하게 변경)
     - 비로그인 유저인 경우:
       1. `e.preventDefault()`로 링크 이동 중단
       2. 사이드바가 열려 있다면 닫음 (`isMenuOpen = false`)
       3. `toast.error("로그인이 필요한 서비스입니다.")` 표시
       4. 콜백 유형 설정 (`authCallback = "mypage"`)
       5. 로그인 모달 엶 (`isAuthOpen = true`)
   - **로그인 성공 핸들러 (`handleLoginSuccess`)**:
     - `authCallback` 값을 확인합니다.
       - `"submit"`인 경우: `isSubmitOpen`을 `true`로 설정하고 `authCallback`을 `null`로 리셋합니다.
       - `"mypage"`인 경우: `router.push("/mypage")`로 이동하고 `authCallback`을 `null`로 리셋합니다.

3. **마크업 구조 개편**
   - **데스크톱 뷰 (`hidden sm:flex`)**:
     - 로고 및 기본 메뉴(지도 홈, 마이페이지)는 항상 노출됩니다.
     - 우측 영역에 "합주실 제보" 버튼이 로그인 여부와 무관하게 상시 노출됩니다.
     - 로그인 상태에 따라 "사용자 정보 + 로그아웃 버튼" 또는 "로그인 / 회원가입" 버튼(`AuthModal`)을 노출합니다.
   - **모바일 뷰 (`sm:hidden`)**:
     - 로고와 햄버거 메뉴 버튼만 노출합니다.
     - 햄버거 버튼 클릭 시 `Sheet`가 열립니다.
     - **사이드바(Sheet) 내용물**:
       - **헤더**: 로고 및 닫기 버튼
       - **로그인 유도/정보 영역**:
         - 로그인 시: 사용자 아이콘 + 닉네임 + 역할 배지 형태의 카드 배치
         - 비로그인 시: "로그인 후 이용해 보세요." 안내 메시지 + 로그인 모달을 트리거하는 버튼 배치
       - **메뉴 링크**: "지도 홈", "마이페이지" 리스트 목록
       - **하단 액션**: "합주실 제보" 및 "로그아웃"(로그인 상태 시) 버튼 배치

## 4. 검증 시나리오

1. **데스크톱 정상 시나리오**:
   - 로그인된 상태에서 "합주실 제보" 클릭 시 제보 폼 모달이 즉시 열리는지 확인합니다.
   - 비로그인 상태에서 "합주실 제보" 클릭 시 토스트 발생 후 로그인 모달이 열리는지 확인합니다.
   - 로그인 모달에서 로그인을 정상 완료했을 때 로그인 모달이 닫히고 제보 폼 모달이 자동으로 뜨는지 확인합니다.
   - 비로그인 상태에서 "마이페이지" 클릭 시 토스트 발생 후 로그인 모달이 열리는지, 로그인 성공 후 마이페이지로 이동하는지 확인합니다.
2. **모바일 정상 시나리오**:
   - 화면 폭을 줄여 `sm` 해상도 이하일 때 햄버거 버튼이 보이고 기존 메뉴들이 숨겨지는지 확인합니다.
   - 햄버거 버튼 클릭 시 우측에서 사이드바가 부드럽게 열리는지 확인합니다.
   - 사이드바 내에서 "합주실 제보" 클릭 시 사이드바가 닫힘과 동시에 화면에 알림 토스트와 로그인 모달이 연이어 뜨는지 검증합니다.
3. **회귀 검증**:
   - `npm run test`를 통해 기존 백엔드/프론트엔드 유닛 테스트가 통과하는지 확인합니다.
   - `npm run build`를 통해 빌드 타입 에러나 린트 에러가 발생하지 않는지 확인합니다.
