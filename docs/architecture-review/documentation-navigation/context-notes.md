# 문서 탐색 구조 정리 컨텍스트 노트

## 결정

- 문서 탐색의 공개 계약은 `AGENTS.md`가 안내하는 구현 계획 파일이 실제 존재하는 것과 핵심 문서에 삽입 줄 번호가 없는 것으로 정의한다.
- 회귀 테스트는 문서 내용을 구현 세부사항처럼 과도하게 고정하지 않고 경로 존재 여부와 손상 패턴만 검증한다.
- 구현 계획 문서의 중복 링크 섹션은 첫 번째 손상된 복사본만 제거하고 정상 섹션은 유지한다.
- `docs/adr/` 부재는 `docs/agents/domain.md` 지침에 따라 문제로 제안하거나 빈 폴더를 만들지 않는다.

## 진행 기록

- GitHub 이슈 `#5`를 생성했다.
- 브랜치 `issue-5-documentation-navigation`을 생성했다.
- `CONTEXT.md` 9~25행과 구현 계획 문서 7~9행에서 삽입 줄 번호를 확인했다.
- 구현 계획 경로 존재 여부 테스트가 잘못된 `AGENTS.md` 경로를 검출하는 RED를 확인한 뒤 실제 경로로 수정해 GREEN을 확인했다.
- 삽입 줄 번호 테스트가 `CONTEXT.md` 손상을 검출하는 RED를 확인한 뒤 두 핵심 문서에서 손상 접두사를 제거해 GREEN을 확인했다.
- 구현 계획 문서에서 손상된 중복 링크 섹션을 제거하고 섹션이 한 번만 존재하는지 검증을 추가했다.
- 담당 문서 회귀 테스트 3개가 통과했다.
- `npm run build`는 네트워크 권한으로 Google Fonts를 가져온 뒤 통과했다.
- `npm test -- --run`은 다른 에이전트의 진행 중인 변경에서 2건 실패했다. `canTransitionStudioUpdateRequest is not a function`과 수정 요청 테스트의 예상 `400`, 실제 `404` 불일치이며 담당 문서 변경과 무관하다.
- 공용 작업 트리의 현재 브랜치가 다른 에이전트에 의해 변경되어, 다른 변경을 포함하지 않도록 담당 파일만 `issue-5-documentation-navigation` 브랜치에 선별 커밋한다.
